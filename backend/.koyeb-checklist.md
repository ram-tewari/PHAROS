# Koyeb Deployment Checklist

Use this checklist to ensure a smooth deployment to Koyeb Free Tier.

## Pre-Deployment Checklist

### External Services Setup

- [ ] **NeonDB (PostgreSQL)**
  - [ ] Account created at https://neon.tech
  - [ ] Project created: `pharos-db`
  - [ ] Connection string copied
  - [ ] Format verified: `postgresql://user:password@host/database?sslmode=require`

- [ ] **Upstash (Redis)**
  - [ ] Account created at https://upstash.com
  - [ ] Database created: `pharos-cache`
  - [ ] Connection string copied
  - [ ] Format verified: `rediss://default:password@host:6379`

- [ ] **Koyeb Account**
  - [ ] Account created at https://www.koyeb.com
  - [ ] Email verified
  - [ ] Dashboard accessible

### Repository Setup

- [ ] **Deployment Files**
  - [ ] `backend/Dockerfile` exists
  - [ ] `backend/entrypoint.sh` exists
  - [ ] `backend/koyeb.yaml` exists
  - [ ] `backend/config/requirements-cloud.txt` exists
  - [ ] `backend/config/requirements-base.txt` exists
  - [ ] `backend/gunicorn_conf.py` exists

- [ ] **Git Repository**
  - [ ] Code pushed to GitHub/GitLab
  - [ ] Branch: `main` or `master`
  - [ ] Repository accessible (public or Koyeb has access)

### Configuration Files

- [ ] **Dockerfile**
  - [ ] Multi-stage build configured
  - [ ] Python 3.11-slim base image
  - [ ] Dependencies installed from requirements-cloud.txt
  - [ ] Non-root user created
  - [ ] Health check configured
  - [ ] Port 8000 exposed

- [ ] **entrypoint.sh**
  - [ ] Shebang line: `#!/bin/bash`
  - [ ] Error handling: `set -e`
  - [ ] Environment validation
  - [ ] Alembic migration command
  - [ ] Gunicorn startup command
  - [ ] Executable permissions (Linux/Mac: `chmod +x`)

- [ ] **koyeb.yaml**
  - [ ] Service name: `pharos-api`
  - [ ] Build type: `docker`
  - [ ] Dockerfile path: `backend/Dockerfile`
  - [ ] Build context: `backend`
  - [ ] Region: `fra` (or preferred)
  - [ ] Instance type: `nano`
  - [ ] Port: `8000`
  - [ ] Health check: `/health`
  - [ ] Environment variables configured
  - [ ] Secrets referenced (not hardcoded)

## Deployment Checklist

### Step 1: Create Secrets

- [ ] **database-url**
  - [ ] Created in Koyeb Dashboard or CLI
  - [ ] Value: PostgreSQL connection string
  - [ ] SSL mode enabled: `?sslmode=require`

- [ ] **redis-url**
  - [ ] Created in Koyeb Dashboard or CLI
  - [ ] Value: Redis connection string
  - [ ] TLS enabled: `rediss://` protocol

- [ ] **pharos-api-key**
  - [ ] Created in Koyeb Dashboard or CLI
  - [ ] Value: Secure random key (32+ characters)
  - [ ] Stored securely (password manager)

### Step 2: Deploy Service

#### Option A: Koyeb Dashboard

- [ ] Navigate to https://app.koyeb.com/services
- [ ] Click "Create Service"
- [ ] Select "Docker" build method
- [ ] Configure Git repository
  - [ ] Repository URL entered
  - [ ] Branch selected: `main`
- [ ] Configure build
  - [ ] Dockerfile: `backend/Dockerfile`
  - [ ] Build context: `backend`
- [ ] Configure instance
  - [ ] Region: `fra` (Frankfurt)
  - [ ] Instance type: `nano` (512MB)
  - [ ] Scaling: min=1, max=1
- [ ] Configure port
  - [ ] Port: `8000`
  - [ ] Protocol: `HTTP`
  - [ ] Public: `Yes`
- [ ] Configure health check
  - [ ] Path: `/health`
  - [ ] Port: `8000`
  - [ ] Protocol: `HTTP`
  - [ ] Interval: `30s`
  - [ ] Timeout: `10s`
  - [ ] Grace period: `60s`
- [ ] Add environment variables
  - [ ] `ENVIRONMENT=production`
  - [ ] `PORT=8000`
  - [ ] `WEB_CONCURRENCY=2`
  - [ ] `LOG_LEVEL=info`
  - [ ] `JSON_LOGGING=true`
  - [ ] `TZ=UTC`
- [ ] Link secrets
  - [ ] `DATABASE_URL` → `database-url`
  - [ ] `REDIS_URL` → `redis-url`
  - [ ] `PHAROS_API_KEY` → `pharos-api-key`
- [ ] Click "Deploy"

#### Option B: Koyeb CLI

- [ ] Install Koyeb CLI
- [ ] Login: `koyeb login`
- [ ] Create secrets:
  ```bash
  koyeb secrets create database-url --value "postgresql://..."
  koyeb secrets create redis-url --value "rediss://..."
  koyeb secrets create pharos-api-key --value "your-key"
  ```
- [ ] Deploy service:
  ```bash
  cd backend
  koyeb service create --config koyeb.yaml
  ```

### Step 3: Monitor Deployment

- [ ] **Build Phase**
  - [ ] Build logs visible
  - [ ] Dependencies installing
  - [ ] No build errors
  - [ ] Image created successfully

- [ ] **Deployment Phase**
  - [ ] Container starting
  - [ ] Migrations running
  - [ ] Server starting
  - [ ] Health check passing

- [ ] **Service Status**
  - [ ] Status: `Running`
  - [ ] Health: `Healthy`
  - [ ] URL generated: `https://pharos-api-*.koyeb.app`

## Post-Deployment Checklist

### Verification

- [ ] **Health Check**
  ```bash
  curl https://your-app.koyeb.app/health
  ```
  - [ ] Status: `200 OK`
  - [ ] Response: `{"status": "healthy", ...}`
  - [ ] Database: `connected`
  - [ ] Redis: `connected`

- [ ] **API Documentation**
  ```bash
  curl https://your-app.koyeb.app/docs
  ```
  - [ ] Swagger UI loads
  - [ ] All endpoints visible
  - [ ] Try it out works

- [ ] **Test Endpoint**
  ```bash
  curl -X POST https://your-app.koyeb.app/api/context/retrieve \
    -H "Content-Type: application/json" \
    -H "X-API-Key: your-api-key" \
    -d '{"query": "test", "codebase": "test", "max_chunks": 5}'
  ```
  - [ ] Status: `200 OK` or `404` (if no data)
  - [ ] No 500 errors
  - [ ] Response time < 2s

### Monitoring

- [ ] **Koyeb Dashboard**
  - [ ] Metrics visible: https://app.koyeb.com/services/pharos-api/metrics
  - [ ] Logs streaming: https://app.koyeb.com/services/pharos-api/logs
  - [ ] Events recorded: https://app.koyeb.com/services/pharos-api/events

- [ ] **Resource Usage**
  - [ ] CPU usage < 80%
  - [ ] Memory usage < 400MB (80% of 512MB)
  - [ ] No OOM errors in logs

- [ ] **Database Connections**
  - [ ] NeonDB connections < 20 (out of 22 max)
  - [ ] No connection pool exhaustion errors

- [ ] **Redis Cache**
  - [ ] Upstash requests < 10K/day (free tier limit)
  - [ ] Cache hit rate > 50%

### Performance

- [ ] **Response Times** (check logs or metrics)
  - [ ] Health check: < 100ms
  - [ ] CRUD operations: < 200ms
  - [ ] Search: < 500ms
  - [ ] Context retrieval: < 1s
  - [ ] Pattern learning: < 2s

- [ ] **Throughput**
  - [ ] Handles 10+ concurrent requests
  - [ ] No request timeouts
  - [ ] No worker crashes

### Security

- [ ] **Secrets**
  - [ ] No secrets in logs
  - [ ] No secrets in environment variables (use Koyeb Secrets)
  - [ ] No secrets in Git repository

- [ ] **HTTPS**
  - [ ] All requests use HTTPS
  - [ ] SSL certificate valid
  - [ ] No mixed content warnings

- [ ] **API Key**
  - [ ] API key required for sensitive endpoints
  - [ ] Invalid API key returns 401 Unauthorized
  - [ ] API key not logged

## Troubleshooting Checklist

### Build Failures

- [ ] **"Cannot find Dockerfile"**
  - [ ] Verify path: `backend/Dockerfile`
  - [ ] Check build context: `backend`

- [ ] **"pip install failed"**
  - [ ] Verify `requirements-cloud.txt` exists
  - [ ] Verify `requirements-base.txt` exists
  - [ ] Check for syntax errors in requirements

- [ ] **"Permission denied: entrypoint.sh"**
  - [ ] Run: `chmod +x backend/entrypoint.sh`
  - [ ] Commit and push changes

### Runtime Failures

- [ ] **"DATABASE_URL not set"**
  - [ ] Verify secret created: `koyeb secrets list`
  - [ ] Verify secret linked in service config
  - [ ] Check secret value format

- [ ] **"Health check failed"**
  - [ ] Check logs: `koyeb logs pharos-api --tail 100`
  - [ ] Verify migrations ran successfully
  - [ ] Verify server started on port 8000
  - [ ] Test health endpoint manually

- [ ] **"Out of memory (OOM)"**
  - [ ] Reduce workers: `WEB_CONCURRENCY=1`
  - [ ] Check memory usage in metrics
  - [ ] Consider upgrading to micro instance

- [ ] **"Connection timeout"**
  - [ ] Check DATABASE_URL is correct
  - [ ] Check REDIS_URL is correct
  - [ ] Verify NeonDB is not suspended
  - [ ] Verify Upstash is accessible

### Performance Issues

- [ ] **Slow response times**
  - [ ] Check NeonDB autosuspend (adds 10s cold start)
  - [ ] Check Redis cache hit rate
  - [ ] Check worker count (may need more)
  - [ ] Check database query performance

- [ ] **High memory usage**
  - [ ] Reduce worker count
  - [ ] Check for memory leaks in logs
  - [ ] Restart service: `koyeb service restart pharos-api`

- [ ] **High CPU usage**
  - [ ] Check for infinite loops in logs
  - [ ] Check for expensive queries
  - [ ] Consider upgrading instance type

## Rollback Checklist

### Emergency Rollback

- [ ] **Pause service**
  ```bash
  koyeb service pause pharos-api
  ```

- [ ] **Identify issue**
  - [ ] Check logs for errors
  - [ ] Check metrics for anomalies
  - [ ] Identify root cause

- [ ] **Fix issue**
  - [ ] Revert code changes
  - [ ] Fix configuration
  - [ ] Update secrets if needed

- [ ] **Resume service**
  ```bash
  koyeb service resume pharos-api
  ```

### Rollback to Previous Deployment

- [ ] **List deployments**
  ```bash
  koyeb deployment list --service pharos-api
  ```

- [ ] **Rollback**
  ```bash
  koyeb service rollback pharos-api --deployment <id>
  ```

- [ ] **Verify rollback**
  - [ ] Check service status
  - [ ] Test health endpoint
  - [ ] Monitor logs
